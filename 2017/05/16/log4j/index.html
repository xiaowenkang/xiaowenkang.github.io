log4j.properties 配置详解

发表于 2017-05-16 | 分类于 java

简介

log4j有三个主要的组件：Loggers(记录器)，Appenders (输出源)和Layouts(布局)。

Loggers

Loggers分为5个级别：DEBUG、INFO、WARN、ERROR和FATAL。

级别高低顺序：

DEBUG < INFO < WARN < ERROR < FATAL

当Loggers级别设定为INFO时，只会输出级别高于或等于INFO的日志信息，比如，WARN,而不会输出DEBUG.

Appenders

Appenders允许把日志输出到不同的地方，如控制台（Console）,文件（Files）等。
常使用的类有：

  • org.apache.log4j.ConsoleAppender（控制台）
  • org.apache.log4j.FileAppender（文件）
  • org.apache.log4j.DailyRollingFileAppender（每天产生一个日志文件）
  • org.apache.log4j.RollingFileAppender（文件大小到达指定尺寸的时候产生一个新的文件）
  • org.apache.log4j.WriterAppender（将日志信息以流格式发送到任意指定的地方）

Layouts

Layouts能将日志输出格式化，提供了四种日志输出样式.

  • org.apache.log4j.HTMLLayout（以HTML表格形式布局）
  • org.apache.log4j.PatternLayout（可以灵活地指定布局模式）
  • org.apache.log4j.SimpleLayout（包含日志信息的级别和信息字符串）
  • org.apache.log4j.TTCCLayout（包含日志产生的时间、线程、类别等信息）

配置详解

log4j支持两种配置文件格式，一种是XML格式的文件，一种是properties属性文件。下面log4j.properties为例讲解

log4j.properties一般放在classpath路径下
如果采用自动加载的模式，log4j.properties要放在src目录或是源文件目录下。这种方式打包，log4j文件是打进包里面的。

如果手动加载，需要加上如下代码，自己设置文件路径

PropertyConfigurator.configure(“config/log4j.properties”)

这种方式打包更灵活，log4j文件不用打进包里，可以放在jar包外面，如上面的，就是放在和jar文件同目录的config目录里面

配置根Logger
log4j.rootLogger = [ level ] , appenderName1, appenderName2, …
level 表示级别
appenderName：就是指定日志信息要输出到哪里。可以同时指定多个输出目的地，用逗号隔开。

配置日志输出目的地
log4j.appender.appenderName = className
appenderName：自定义appenderName，在log4j.rootLogger中设置过的；
className：可设值如下：

  • org.apache.log4j.ConsoleAppender（控制台）
  • org.apache.log4j.FileAppender（文件）
  • org.apache.log4j.DailyRollingFileAppender（每天产生一个日志文件）
  • org.apache.log4j.RollingFileAppender（文件大小到达指定尺寸的时候产生一个新的文件）
  • org.apache.log4j.WriterAppender（将日志信息以流格式发送到任意指定的地方）

配置日志信息的输出格式（Layout）
log4j.appender.appenderName.layout=className
className：可设值如下：

  • org.apache.log4j.HTMLLayout（以HTML表格形式布局）
  • org.apache.log4j.PatternLayout（可以灵活地指定布局模式）
  • org.apache.log4j.SimpleLayout（包含日志信息的级别和信息字符串）
  • org.apache.log4j.TTCCLayout（包含日志产生的时间、线程、类别等等信息）

格式化符号说明

  • %p：输出日志信息的优先级，即DEBUG，INFO，WARN，ERROR，FATAL。
  • %d：输出日志时间点的日期或时间，默认格式为ISO8601，也可以在其后指定格式，如：%d{yyyy/MM/dd HH:mm:ss,SSS}。
  • %r：输出自应用程序启动到输出该log信息耗费的毫秒数。
  • %t：输出产生该日志事件的线程名。
  • %l：输出日志事件的发生位置，相当于%c.%M(%F:%L)的组合，包括类全名、方法、文件名以及在代码中的行数。例如：test.TestLog4j.main(TestLog4j.java:10)。
  • %c：输出日志信息所属的类目，通常就是所在类的全名。
  • %M：输出产生日志信息的方法名。
  • %F：输出日志消息产生时所在的文件名称。
  • %L:：输出代码中的行号。
  • %m:：输出代码中指定的具体日志信息。
  • %n：输出一个回车换行符，Windows平台为”\r\n”，Unix平台为”\n”。
  • %x：输出和当前线程相关联的NDC(嵌套诊断环境)，尤其用到像java servlets这样的多客户多线程的应用中。
  • %%：输出一个”%”字符。

另外，还可以在%与格式字符之间加上修饰符来控制其最小长度、最大长度、和文本的对齐方式。如：
1)%20c：指定输出category的名称，最小的长度是20，如果category的名称长度小于20的话，默认的情况下右对齐。
2)%-20c：”-“号表示左对齐。
3)%.30c：指定输出category的名称，最大的长度是30，如果category的名称长度大于30的话，就会将左边多出的字符截掉，但小于30的话也不会补空格。

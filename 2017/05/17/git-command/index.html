git 配置

设置自己的git信息

git config --global user.name "username" （此处name可修改也不是用于登录github的登录名）

git config --global user.email "your_email@youremail.com"

git config --list (查看自己git信息)

1. 查看是否存在SSH密钥

cd ~/.ssh

2. 创建新的ssh key

ssh-keygen -t rsa -C "your_email@youremail.com"

3. 复制ssh key到github

On the GitHub site Click “Account Settings” > Click “SSH Keys” > Click “Add SSH key”

4. 测试ssh连接github

ssh -T git@github.com

查看连接详细信息

ssh -v git@github.com

git 命令

git init 初始化目录为git仓库

git status 查看仓库当前状态

git diff 查看difference

git add git commit 两步提交仓库

git diff
是工作区(work dict)和暂存区(stage)的比较

git diff –cached
是暂存区(stage)和分支(master)的比较

git diff HEAD
比较工作区和版本库最新版本的区别

git reset 版本回退，既可以回退版本，也可以把暂存区的修改回退到工作区

git checkout -- file 丢弃工作区的修改

git remote add origin git@github.com:xiaowenakng/XXX.git
关联远程库

远程库的名字就是origin

git push -u origin master 第一次推送本地master分支的所有内容

git push origin master

svn版本控制系统创建和切换分支很慢，分支功能成为摆设
